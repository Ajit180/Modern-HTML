
Question 4: List any 3 Ways which help us in improving the accessiblity of HTML.

Solution :  
  
       Improving the accessibility of HTML involves adopting practices and techniques that ensure web content is usable and understandable by a diverse audience, including individuals with disabilities. Here are three ways to enhance HTML accessibility:

1. Semantic HTML:
   - Use semantic HTML elements to convey the structure and meaning of content. Semantic elements like `<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<footer>`, and others provide a clear and meaningful structure to web documents. Assistive technologies can better interpret and present content when semantic HTML is used, improving the overall accessibility of the page.

2. Proper Use of ARIA (Accessible Rich Internet Applications) Attributes:
   - ARIA attributes enhance the accessibility of dynamic content and web applications. When using interactive elements like sliders, tabs, or custom widgets, include ARIA attributes to provide additional information to assistive technologies. For example, `role`, `aria-label`, `aria-labelledby`, and `aria-describedby` can be used to give context and improve the understanding of complex user interface components.

3. Accessible Forms:
   - Ensure that forms are accessible to users with various abilities. Use the `<label>` element to associate labels with form controls explicitly. Provide meaningful descriptions using the `aria-label` attribute or associated `<label>` elements for form controls that may lack visible labels. Use appropriate input types (e.g., `email`, `tel`, `date`) to enable better form validation and assistive technology support. Additionally, use the `title` attribute for additional context when needed.
